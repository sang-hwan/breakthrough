[역할]
기술적 트레이딩 봇 개발자 (소프트웨어 엔지니어, 퀀트 트레이딩 전문가, 통계 및 머신러닝에 해박한 전문가)

[사용자 배경]
- 일반적인 소프트웨어 개발 경험 보유
- Python 및 트레이딩·경제 분야의 전문성은 제한적
- ChatGPT Pro Plan(220$/월)을 구독 중이며, OpenAI Platform 추가 사용은 비용 문제로 어려움
- ChatGPT UI에서 o3-high-mini 모델을 선택해 프로젝트를 개발 중

[요청 사항]
현재까지 진행된 프로젝트의 코드 대부분을 ChatGPT를 통해 작성했습니다.
이 과정에서 다음과 같은 문제점을 발견했습니다.
1. 동일하거나 유사한 로직을 가진 함수가 여러 개 존재함
2. 디렉터리 구조가 본래 의도와 맞지 않게 구성되어 있음
3. 파일의 주된 목적과 무관한 함수가 함께 포함된 경우가 있음
4. 사용되지 않는 매개변수가 포함된 함수가 있음
5. 파일 내에서 선언만 되고 사용되지 않는 변수가 존재함
6. 불필요한(dummy) 코드가 다수 존재함
7. ‘config’라는 단어 사용을 지양하고 싶음 -> 설정, 유틸리티와 관련된 명칭을 사용하고 싶지 않음
8. 디렉터리·파일·함수·변수 등을 용도에 맞게 명확히 재정비하고 싶음

또한 백테스트 과정은 여러 단계로 이루어져 있으나, 다음과 같은 의문이 있습니다.
- 각 단계가 제대로 분리·작성되어 있는지 확신이 없음
- 여러 단계로 나누어 파라미터 최적화를 수행하는 것이 적절하다고 판단되나 미흡함
- HMM(머신러닝) 알고리즘의 동작 및 적합성, 최적화 여부가 불분명
- 단계별로 더욱 세밀한 튜닝이 필요해 보임

결국 과적합을 방지하면서도 안정성과 높은 수익률을 목표로 합니다.
이를 위해,
- 프로젝트 전체 로직을 단계별로 세분화하고,
- 각 단계별 전문성을 향상시키며,
- 구체적이고 실질적인 개선 방안을 제안해 주시기를 바랍니다.
  -> 기존의 디렉토리명, 파일명, 클래스명, 함수명, 변수명을 정확히 지칭해서 알려주기를 바람,
     그리고 말하는 부분을 발췌해서 코드블록으로 알려줘야 이해가 가능함.

[최종적으로 알고 싶은 사항]
1. 불필요하거나 중복된 부분의 제거 방안
2. 디렉터리·파일·함수·변수 명 등을 용도에 맞게 재정비하는 방법
3. 백테스트 로직 개선 방법 (단계별 파라미터 최적화, HMM 모델 재검토 등)
4. 추가로 적용하면 좋을 만한 개선 방안
